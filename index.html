Linux学习（一）---常用命令

发表于 2019-05-06

基本命令

1：命令pwd 查询当前文件的绝对路径
2: 命令cat 查看文本文件
3: 命令mkdir 目录下新建一个文件夹,如：在/home/wwwroot目录下新建一个sinozzz123的文件夹，mkdir /home/wwwroot/sinozzz123
4: 命令cp 复制文件或者目录，如：把/home/wwwroot/sinozzz里面的文件和文件夹等复制到home/wwwroot/sinozzz123目录下cp -rf /home/wwwroot/sinozzz/* /home/wwwroot/sinozzz123 r表示目录的递归复制，f表示强制执行。
5： 命令mv 剪切文件或者目录,如需要将/home/music/目录下的1.mp3文件剪切到/home/abc目录下，执行下面的命令即可：# mv /home/music/1.mp3 /home/abc。注：在Linux下，移动文件跟重命名都是同一个命令，mv（就是move的意思）
命令： mv 源文件名 目标文件名

6：命令rm,删除文件或者目录,rm -rf 目录名字-r 就是向下递归，不管有多少级目录，一并删除。-f 就是直接强行删除，不作任何提示的意思。
7: 命令 tar, 解压压缩包，解压很简单 ：
|.tar.gz | 格式解压为 | tar -zxvf xx.tar.gz –C 解压到指定目录 |
|.tar.bz2 | 格式解压为 | tar -jxvf xx.tar.bz2|
其中：-x ： 解压缩压缩档案的参数；
-z ： 是否需要用gzip压缩；
-v： 压缩的过程中显示档案；
-f： 置顶文档名，在f后面立即接文件名，不能再加参数

1.1 Hexo与GitHub的的搭建

详细教程点击查看1和点击查看2
下面我讲一讲域名和GitHub Page的绑定，域名申请可以去阿里云和腾讯云
申请好域名之后：
第一步：在命令窗口ping一下：ping dcl8023.github.io得到如下图，说明可以链接到GitHub服务器，得到ip:185.199.109.153

第二步：添加要解析的域名，对于记录类型为A（将域名指向一个IPv4地址）分别要添加主机记录为www和@两个记录，记录值中添加刚才获得的ip:185.199.109.153 如图：，据说如果GitHub改了IP之后就无法访问了，不稳定。而记录类型为CNAME的，也分别要添加主机记录为www和@两个记录，记录值中添加 username.github.io.,如图：
第三步：在根目录source下新建一个README.md文件，将自己申请的域名复制进去
第四步：去GitHub你刚刚新建的库里设置，添加申请的域名，
第五步：ping域名:ping:www.dcpolar.xyz,成功说明解析成功
然后hexo clean ->hexo g ->hexo d

1.2 Hexo的美化

美化教程参照reuixiy的美化教程和JuLi距离的教程

1.2.1 在Hexo博客上添加可爱的Live 2D模型

1：live2d-widget-model-haru模型需要安装安装模块:npm install --save hexo-helper-live2d
试试 yarn add hexo-helper-live2d, 据说高能的 yarn 会比垃圾 npm 少很多麻烦呢。 Yarn
2:官方提供了多款精美的模型，点击查看
你需要先使用 npm install 模型的包名 来安装，然后将包名输入位于_config.yml 的 model.use 中
3：配置内容可以直接配置在 Hexo 的配置文件 _config.yml中，也可以配置在主题的Hexo根目录/themes/主题目录/_config.yml 文件中，但推荐配置在 Hexo 的配置文件中，防止更换主题后 live2d 失效。复制下面代码：

enable: true  
scriptFrom: local
model: 
    use: live2d-widget-model-haruto #已经安装的模型选择
display: 
    position: right  #模型位置
    width: 300       #模型宽度
    height: 600      #模型高度
mobile: 
    show: true     #是否在手机端显示

2 MarkDown 与 文章编辑

文章配置

在hexo文件夹下右键点击Git Base here后键入$hexo new page "name"，source/_post 文件夹中生成name.md文件，和一个name的文件夹用来存储你的图片，将你要添加的图片放在这个文件夹下面，要想在博客上显示图片，在你的hexo目录下执行这样一句话npm install hexo-asset-image --save，这是下载安装一个可以上传本地图片的插件.之后你需要修改配置文件，把主页配置文件_config.yml里的post_asset_folder:这个选项设置为true，事先准备公国完成后就可以打开name.md文件编辑。MarkDown教程链接：
http://www.runoob.com/markdown/md-tutorial.html 或者用这种形式：[链接名称](链接地址) Markdown 教程

2.2 Markdown 段落

换行

段落的换行是使用两个以上空格加上回车。

删除线

如果段落上的文字要添加删除线，在两端加上~~不知道为什么不行，另一种方法只需要在文字的两端加上两个 <del> 即可，实例如下：<del>我想删除的一段话<del>，显示效果如下所示：我想删除的一段话。

下划线

下划线可以通过 HTML的<u>标签来实现<u>带下划线文本</u>显示效果如下所示：带下划线文本

附注：尽量不要给文本加下划线，因为这会和超链的表现形式混淆，会被误以为是个超链。

Markdown 代码

如果是段落上的一个函数或片段的代码可以用反引号把它包起来（），例如:printf()`
代码块：代码之间分别用三个反引号包起来，且两边的反引号单独占一行,显示结果如下：

Markdown 图片

Markdown 图片语法格式如下：![avatar](图片地址) ![你想输入的替代文字](xxxx/图片名.jpg) 两个都是一样的
开头一个感叹号 !接着一个方括号，里面放上图片的替代文字接着一个普通括号，里面放上图片的网址，最后还可以用引号包住并加上选择性的 ‘title’ 属性的文字。

插入本地图片

只需要在基础语法的括号中填入图片的位置路径即可，支持绝对路径和相对路径。

Markdown 制作表格使用 | 来分隔不同的单元格，使用 - 来分隔表头和其他行。在一句话前面加>,表示引用，语法格式如下：
